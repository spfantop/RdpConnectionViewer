# RDP 连接记录查看工具

## 一、工具简介

**RDP 连接记录查看工具** 是一个用于 Windows 服务器 / Windows 远程桌面的轻量级管理工具。

它可以帮助管理员快速查看本机远程桌面连接记录，包括：

* RDP 认证成功记录
* 远程桌面登录成功记录
* 会话断开记录
* 会话重新连接记录
* 会话注销记录
* 来源 IP 统计
* 可疑 IP 封禁 / 解封
* RDP 端口查看与修改
* 本地远程账号密码管理
* Windows 防火墙封禁诊断

本工具主要适合用于：

* 检查是否有人远程连接过服务器
* 查看远程连接来源 IP
* 排查 RDP 被爆破尝试的问题
* 快速封禁异常 IP
* 修改远程桌面端口
* 管理远程桌面登录账号

解压工具包后，运行：

```bat
build_exe.bat
```


以后直接运行 `RdpConnectionViewer.exe` 即可。

```text

```

可根据该日志排查启动或运行错误。

## 四、主要功能说明

### 1. 查看 RDP 连接记录

工具会读取 Windows 事件日志中的远程桌面相关记录，并整理为表格显示。

常用事件包括：

| 事件 ID   | 含义        |
| ------- | --------- |
| 1149    | 远程桌面认证成功  |
| 21      | 会话登录成功    |
| 22      | Shell 启动  |
| 23      | 会话注销成功    |
| 24      | 会话断开      |
| 25      | 会话重新连接    |
| 39 / 40 | 会话断开原因    |
| 4624    | 登录成功，安全日志 |
| 4625    | 登录失败，安全日志 |

默认情况下，工具优先读取 RDP 专用日志，速度较快。

如果需要查看登录失败记录，可以勾选：

```text
包含安全日志 4624 / 4625
```

注意：安全日志通常较大，读取速度可能明显变慢。

---

### 2. 筛选连接记录

支持按以下条件筛选：

* 最近天数
* 最大读取条数
* 来源 IP
* 用户名
* 是否包含安全日志
* 是否检查封禁状态

推荐日常使用参数：

```text
最近天数：7
最大条数：500
不勾选安全日志
不勾选检查封禁状态
```

这样速度较快，占用较低。

---

### 3. 来源 IP 统计

工具会按照来源 IP 汇总连接记录，包括：

* 总连接次数
* 认证成功次数
* 登录成功次数
* 登录失败次数
* 断开次数
* 重新连接次数
* 涉及用户
* 是否被封禁

该功能适合快速发现异常访问来源，例如短时间内大量失败登录的 IP。

---

### 4. 封禁 IP

工具支持封禁指定 IP。

封禁逻辑基于 Windows Defender 防火墙规则实现，会创建针对指定来源 IP 的阻止规则。

新版封禁策略包括：

* 入站阻止该 IP
* 出站阻止该 IP
* 阻止当前 RDP 端口
* 阻止默认 RDP 端口 3389
* 同时处理 TCP / UDP

封禁后建议配合：

```text
断开此会话
```

因为已经建立的远程桌面连接不一定会被防火墙规则立即踢下线。

正确测试方式：

```text
1. 封禁目标 IP
2. 断开该 IP 当前会话
3. 让该 IP 重新连接测试
```

如果封禁后原有远程桌面窗口仍然能操作，不一定代表封禁失败，可能只是已有连接尚未断开。

---

### 5. 解封 IP

工具支持解封之前由本工具创建的封禁规则。

解封时会删除规则名前缀为：

```text
RDPViewer_Block_
```

的相关防火墙规则。

工具不会主动删除用户手动创建的其他防火墙规则。

---

### 6. 封禁当前活动 RDP 远端 IP

如果事件日志中的来源 IP 不准确，或者服务器经过 NAT、端口转发、堡垒机、RD Gateway，日志里的 IP 可能不是实际 TCP 连接来源。

此时可以使用：

```text
封禁当前活动 RDP 远端 IP
```

该功能会直接从当前 RDP 网络连接中读取真实远端地址，并对其执行封禁。

---

### 7. 防火墙诊断

如果封禁 IP 后仍能访问，可以使用诊断功能。

工具会生成：

```text
RdpConnectionViewer_firewall_diagnose.txt
```

诊断内容通常包括：

* Windows 防火墙是否开启
* 当前防火墙 Profile 状态
* 工具创建的封禁规则
* 当前 RDP 端口
* 当前活动 RDP 连接
* 当前远端连接 IP
* 可能导致封禁无效的原因

常见封禁无效原因：

1. Windows Defender 防火墙未开启；
2. 当前连接是已建立连接，未被立即断开；
3. 封禁的 IP 不是实际远端 IP；
4. 服务器前面有 NAT、端口映射、堡垒机或 RD Gateway；
5. 第三方安全软件接管了防火墙；
6. 当前使用的不是 Windows 防火墙规则控制流量；
7. RDP 服务端口不是 3389，但只封禁了错误端口；
8. 连接通过内网代理或网关转发，服务器看到的是网关 IP。

## 五、RDP 端口管理

工具支持查看和修改当前 RDP 端口。

默认 RDP 端口为：

```text
3389
```

修改端口后，工具会尝试自动添加新端口的防火墙放行规则。

注意：

```text
修改 RDP 端口后，通常需要重启远程桌面服务或重启系统才会生效。
```

如果你正在通过远程桌面连接这台服务器，不建议直接重启远程桌面服务，否则可能会断开当前连接。

更安全的操作建议：

```text
1. 先确认服务器有其他登录方式，例如控制台、VNC、云厂商网页终端；
2. 修改 RDP 端口；
3. 确认新端口防火墙已放行；
4. 重启服务器；
5. 使用新端口连接测试。
```

## 六、账号与密码管理

工具支持对本地 Windows 账号进行部分管理：

* 修改本地账号密码
* 重命名本地账号
* 将账号加入远程桌面用户组

注意：

1. 只能管理本机本地账号；
2. 域账号、Microsoft 在线账号不一定适用；
3. 修改当前正在使用的账号密码前，请确保记住新密码；
4. 重命名账号后，远程登录用户名也会发生变化；
5. 不建议删除或随意修改系统内置管理员账号。

## 七、安全建议

为了降低 RDP 被扫描和爆破的风险，建议配合以下措施使用：

### 1. 修改默认 RDP 端口

默认端口 `3389` 容易被扫描，可以改为其他高位端口。

### 2. 使用强密码

远程桌面账号应避免使用：

```text
admin
administrator
test
user
123456
password
```

建议密码至少包含：

* 大写字母
* 小写字母
* 数字
* 特殊符号
* 长度 12 位以上

### 3. 限制允许访问的 IP

如果远程连接来源固定，建议只允许指定 IP 访问 RDP 端口。

### 4. 开启网络级别身份验证 NLA

建议开启：

```text
仅允许运行使用网络级别身份验证的远程桌面的计算机连接
```

### 5. 定期检查登录失败记录

如果发现某个 IP 有大量失败登录记录，应及时封禁并检查账号安全。

### 6. 不要暴露不必要的远程端口

如果服务器不需要公网远程桌面，建议关闭公网 RDP，只通过 VPN、内网或堡垒机访问。

## 八、性能说明

本工具设计目标是轻量、低占用、手动刷新。

默认情况下：

* 不开启后台轮询；
* 不定时自动扫描；
* 不读取完整安全日志；
* 不全量扫描所有防火墙规则；
* 只在点击刷新时读取事件日志；
* 默认限制最大记录数，避免日志过多导致卡顿。

如果感觉读取慢，可以尝试：

```text
1. 减少最近天数；
2. 降低最大条数；
3. 不勾选安全日志；
4. 不勾选检查封禁状态；
5. 只在需要时使用封禁诊断。
```

## 九、常见问题

### 1. 为什么封禁 IP 后还能远程操作？

可能是已经建立的 RDP 会话还没有被断开。

建议：

```text
先封禁 IP，再断开此会话，然后重新连接测试。
```

---

### 2. 为什么封禁 IP 后重新连接仍然能连？

请检查：

```text
1. Windows 防火墙是否开启；
2. 封禁的 IP 是否是真实远端 IP；
3. 是否经过 NAT、堡垒机、RD Gateway；
4. 是否有第三方安全软件覆盖 Windows 防火墙；
5. 是否使用了其他端口或其他远程方式。
```

可以运行工具内的防火墙诊断功能查看详情。

---

### 3. 为什么来源 IP 显示为空？

部分 Windows 事件本身不携带来源 IP。

工具会尝试通过以下方式补全：

```text
1. 读取 1149 认证日志；
2. 读取会话日志；
3. 根据 SessionID 关联；
4. 根据活动 RDP 连接补全。
```

但如果日志中本身没有记录，仍可能显示为空。

---

### 4. 为什么读取安全日志很慢？

`Security` 日志通常非常大，尤其是服务器运行时间较长时。

建议日常不勾选安全日志，只在排查失败登录、爆破尝试时临时开启。

---

### 5. 为什么修改 RDP 端口后无法连接？

可能原因：

```text
1. 新端口防火墙没有放行；
2. RDP 服务没有重启；
3. 云服务器安全组没有放行新端口；
4. 连接时仍然使用旧端口；
5. 路由器端口转发没有同步修改。
```

远程服务器修改端口前，务必确认有备用登录方式。

## 十、免责声明

本工具用于服务器管理员查看和管理本机 RDP 连接记录。

请仅在你拥有管理权限的服务器或电脑上使用。

修改账号、密码、远程桌面端口、防火墙规则等操作可能导致远程连接中断，请谨慎操作。

建议在执行高风险操作前确认：

```text
1. 已备份重要配置；
2. 有备用登录方式；
3. 知道当前服务器公网 IP、内网 IP、远程端口；
4. 明确当前操作影响。
```

因误操作导致无法远程连接、账号无法登录、端口不通等问题，需要通过本地控制台、云厂商控制台、VNC、IPMI、PVE 控制台等方式恢复。
